# Analiso Frontend Skill

Você está trabalhando no frontend da Analiso.

Seu papel é melhorar:
- clareza
- ativação
- hierarquia visual
- confiança
- leitura guiada
- valor percebido

## Quando usar
Use esta skill em tarefas envolvendo:
- dashboard
- watchlist
- explorar
- comparação
- análise da empresa
- onboarding
- CTAs
- filtros
- cards
- tooltips
- empty states
- alertas
- Mapa dos 5 pilares

## Objetivo principal
Transformar informação financeira em experiência de leitura guiada:
- com baixo esforço cognitivo
- com intenção clara
- com contexto verificável
- com progressão de profundidade
- sem cara de portal financeiro ou painel genérico

## Workflow obrigatório
1. Identifique a intenção primária da tela.
2. Identifique a principal dor do usuário nessa tela.
3. Identifique a ação mais importante.
4. Rebaixe ou compacte tudo que competir com essa ação.
5. Verifique se a tela começa com valor ou com ruído/setup.
6. Aplique os recursos desta skill.
7. Antes de concluir, rode os scripts de validação.

## Hard rules
- Não desenhar telas como dashboard genérico de BI.
- Não usar gráfico só porque “fica bonito”.
- Não duplicar a mesma interpretação em múltiplos lugares sem necessidade.
- Não deixar metodologia competir com valor inicial.
- Não deixar filtros dominarem a entrada da página.
- Não tratar onboarding como demo de funcionalidades.
- Não usar vieses para pressionar decisão financeira.
- Não deixar secondary blocks com protagonismo de primary blocks.

Resuma:
- o que foi simplificado
- o que ficou mais claro
- qual dor do usuário foi melhor atacada
- qual conteúdo foi rebaixado/compactado
- o que ainda ficou para depois
